## 1. 项目简介

本工具是一个基于 **Flask Web 框架** 和 **Microsoft Word COM 自动化引擎** 的本地 PDF ↔ Word 双向转换工具。

**核心特点：**
- 所有转换在本地计算机完成，文件不会上传到任何云端服务器
- 使用 Microsoft Word 原生 PDF 重排引擎，实现最高保真度转换
- 提供简洁美观的 Web 界面，操作直观
- 支持拖拽上传、实时进度显示、一键下载

---

## 2. 系统要求

| 要求项 | 说明 |
|---|---|
| 操作系统 | **Windows 10 / Windows 11**（不支持 macOS / Linux） |
| Microsoft Word | **Word 2016 或更高版本**（不支持 WPS Office） |
| Python | Python 3.9 或更高版本 |
| 磁盘空间 | 至少 500MB 可用空间 |
| 内存 | 至少 4GB 可用内存 |

> **⚠️ 重要提示**: 本工具调用 Microsoft Word 的 COM 接口进行转换，因此必须安装正版 Microsoft Word。WPS Office 不支持 COM 自动化接口。

## 3. 快速开始

### 3.1 首次安装

```bash
# 1. 进入项目目录
cd "项目目录"

# 2. 创建 Python 虚拟环境（仅首次）
python -m venv venv

# 3. 激活虚拟环境
venv\Scripts\activate

# 4. 安装依赖
pip install -r requirements.txt
```

### 3.2 启动服务

**方法一（推荐）**：双击 `start_server.bat`  
**方法二**：在命令行中执行

```bash
venv\Scripts\activate
python app.py
```

服务启动后，浏览器访问 **http://127.0.0.1:5000**

### 3.3 停止服务

- 关闭命令行窗口，或
- 在命令行中按 `Ctrl + C`

## 5. 代码说明

### 5.1 `app.py` — 后端核心

整个应用程序的核心文件，包含以下模块：

| 模块 | 行号（大约） | 功能说明 |
|---|---|---|
| 配置区域 | 23-36 | 文件大小限制、允许格式、超时时间、临时文件上限 |
| 文件安全 | 68-106 | 文件扩展名验证、文件头魔数验证、安全保存 |
| 文件清理 | 109-137 | 过期文件清理（1小时）、超额文件清理（10个上限） |
| 任务管理 | 140-179 | 创建/更新/获取转换任务状态 |
| PDF→Word | 185-252 | PDF 转 Word 核心转换逻辑（Word COM） |
| 下划线恢复 | 255-508 | PDF 下划线检测与恢复（PyMuPDF + python-docx） |
| Word→PDF | 511-571 | Word 转 PDF 核心转换逻辑（Word COM） |
| Flask 路由 | 600-820 | HTTP 请求处理（上传、进度查询、下载、文件管理） |
| 入口 | 827-840 | 程序启动入口 |

**关键配置参数：**

```python
MAX_FILE_SIZE = 50 * 1024 * 1024    # 50 MB
MAX_FILE_AGE = timedelta(hours=1)    # 1小时有效期
MAX_TEMP_FILES = 10                  # 每个文件夹最多10个文件
CONVERSION_TIMEOUT = 120             # 转换超时120秒
```

### 5.2 `templates/index.html` — 前端界面

包含完整的 HTML + CSS + JavaScript，实现：
- 双卡片布局（PDF→Word / Word→PDF）
- 拖拽上传 + 点击上传
- 实时进度轮询
- 临时文件管理面板
- 使用说明区域

### 5.3 `static/css/style.css` — 样式表

独立的 CSS 文件，使用 CSS 自定义属性（Design Tokens）实现深色主题。包括：
- 完整的颜色系统
- 动画效果（淡入、滑动、波纹）
- 响应式布局（桌面端 / 平板 / 手机）
- 无障碍访问支持（reduced-motion）

### 5.4 `start_server.bat` — 启动脚本

Windows 批处理脚本，自动激活虚拟环境并启动 Flask 服务，同时打开浏览器。

### 5.5 `build_html.py` — 辅助脚本

开发阶段使用的辅助脚本，用于快速检查模板状态。日常使用无需运行。

### 5.6 `requirements.txt` — 依赖清单

```

```

## 7. 功能特性

### 7.1 PDF → Word 转换

- 使用 Microsoft Word 原生 PDF 重排引擎
- 1:1 精确还原格式（字体、字号、颜色、间距）
- 自动检测并恢复 PDF 中的下划线和填空横线
- 支持多页 PDF

### 7.2 Word → PDF 转换

- 使用 Microsoft Word 原生引擎
- 100% 保持原始排版
- 支持 .doc 和 .docx 格式

### 7.3 文件管理

- **自动清理**：超过 1 小时的旧文件自动删除
- **数量限制**：每个临时文件夹最多保存 10 个文件
- **手动清理**：网页端一键清空所有临时文件
- **文件浏览**：一键在资源管理器中打开临时文件夹

### 7.4 安全机制

- 文件扩展名白名单验证
- 文件头部魔数验证（防止伪造扩展名）
- UUID 重命名（防止路径穿越攻击）
- 本地处理，零云端传输

1. **启动服务** → 双击 `start_server.bat`

## 9. 常见问题与解决方案

### 问题 1：启动时提示「无法启动此程序，因为计算机中丢失 python39.dll」

**原因**：Python 未正确安装，或虚拟环境损坏。

**解决方案**：
```bash
# 删除旧虚拟环境，重新创建
rmdir /s venv
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

---

### 问题 2：转换一直显示「处理中」但不完成

**原因**：后台 Word 进程卡死。

**解决方案**：
1. 打开任务管理器（`Ctrl + Shift + Esc`）
2. 在「详细信息」中查找 `WINWORD.EXE`
3. 结束所有 WINWORD.EXE 进程
4. 重启本工具后重试

---

### 问题 3：提示「Word COM 转换失败」

**可能原因及解决方案**：

| 原因 | 解决方案 |
|---|---|
| Word 未安装 | 安装 Microsoft Word 2016 或更高版本 |
| Word 许可证未激活 | 激活 Word 许可证 |
| 文件损坏 | 尝试用其他 PDF 阅读器打开确认文件正常 |
| 文件过大 | 文件超过 50MB，请压缩后再上传 |
| Word 正在运行其他任务 | 关闭所有 Word 窗口后重试 |
| Word 弹出对话框 | 检查是否有 Word 窗口弹出（如安全警告），关闭后重试 |

---

### 问题 4：无法访问 http://127.0.0.1:5000

**可能原因及解决方案**：

| 原因 | 解决方案 |
|---|---|
| 服务未启动 | 确认命令行窗口仍打开且显示运行信息 |
| 端口被占用 | 修改 `app.py` 中的端口号（如改为 8080） |
| 防火墙阻止 | 允许 Python 通过 Windows 防火墙 |
| 浏览器缓存 | 尝试使用无痕模式或清除缓存 |

---

### 问题 5：PDF 转换后下划线丢失

**原因**：某些 PDF 的下划线用图形线条实现，而非文本下划线格式。

**状态**：本工具已内置下划线检测与恢复引擎。转换时会自动：
1. 扫描 PDF 中的下划线格式标记
2. 检测向量横线（图形下划线）
3. 在生成的 Word 文档中恢复下划线格式

如果仍有丢失，可能是 PDF 中的下划线形式非常特殊，属于已知限制。

---

### 问题 6：Word 转 PDF 后排版错乱

**原因**：Word 文件中使用了特殊字体，而系统中未安装该字体。

**解决方案**：
1. 安装原文档使用的字体
2. 或在 Word 中将字体替换为系统已有字体后再转换

---

### 问题 7：启动脚本闪退

**原因**：虚拟环境未创建或依赖未安装。

**解决方案**：
```bash
# 在项目目录下打开命令行
cd "项目目录"
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

---

### 问题 8：macOS / Linux 用户可以使用吗？

**不可以**。本工具的核心转换功能依赖 Microsoft Word COM 自动化接口，该接口仅存在于 Windows 平台。macOS 和 Linux 上没有等效的 Word COM 接口。

---

### 问题 9：临时文件在哪里？如何手动清理？

- **上传的源文件**：`uploads\` 文件夹
- **转换后的文件**：`conversions\` 文件夹
- **网页操作**：点击页面中的「打开临时文件夹」或「清空所有临时文件」
- **手动清理**：直接删除 `uploads\` 和 `conversions\` 文件夹中的文件即可

---

### 问题 10：如何查看服务日志？

服务日志直接输出在命令行窗口中。如果遇到错误，命令行会显示详细的 Python 异常堆栈信息，请根据堆栈信息排查。

## 10. 技术架构

```
┌──────────────────────────────────────────────┐
│                  浏览器                      │
│  ┌────────────────────────────────────┐      │
│  │   index.html (HTML + CSS + JS)     │      │
│  │   - 拖拽上传                      │      │
│  │   - AJAX 进度轮询                  │      │
│  │   - 一键下载                       │      │
│  └──────────────┬─────────────────────┘      │
└─────────────────┼────────────────────────────┘
                  │ HTTP (Flask)
┌─────────────────┼────────────────────────────┐
│                 ▼                  Windows   │
│  ┌────────────────────────────────────┐      │
│  │         app.py (Flask)             │      │
│  │  路由: /upload /status /download   │      │
│  └──────────┬───────────┬─────────────┘      │
│             │           │                     │
│             ▼           ▼                     │
│  ┌──────────────┐ ┌──────────────┐           │
│  │ PDF → Word   │ │ Word → PDF   │           │
│  │              │ │              │           │
│  │ PyMuPDF 扫描 │ │ Word COM     │           │
│  │   ↓          │ │ 自动化       │           │
│  │ Word COM     │ │   ↓          │           │
│  │ 自动化       │ │ 保存为 PDF   │           │
│  │   ↓          │ │              │           │
│  │ python-docx  │ │              │           │
│  │ 下划线恢复   │ │              │           │
│  └──────────────┘ └──────────────┘           │
│                                             │
│  ┌──────────────┐                           │
│  │  文件管理     │                           │
│  │  - 自动清理   │                           │
│  │  - 数量限制   │                           │
│  │  - 过期删除   │                           │
│  └──────────────┘                           │
└─────────────────────────────────────────────┘
```

**转换流程：**

1. 用户上传文件 → Flask 接收
2. 验证文件格式和大小
3. 创建异步转换任务
4. 后台线程调用 Word COM 接口执行转换
5. 前端轮询任务状态
6. 转换完成 → 用户下载结果文件
7. 系统自动清理过期和超额文件
